# Data Sharing Policy

_Last updated: 16 June 2026_

## 1. Purpose

This policy describes how the data distributed with the **Authenticated Robotic Teleoperation with Task Recognition** repository may be accessed, used, shared, licensed, and cited.

The repository contains experimental data and code for task identification, user identification, and user authentication based on motion patterns captured with wearable inertial sensors.

## 2. Data covered by this policy

This policy applies to all datasets distributed in this repository, including:

- raw inertial sensor recordings in `data/raw/`;
- flattened feature data such as `data/raw/data.csv`;
- label files such as `data/raw/labels.csv`;
- processed data generated by the repository scripts in `data/processed/`.

The raw data include quaternion components for arm joints and end-effector information collected during teleoperation trials. Labels describe experimental metadata such as task, anonymized subject identifier, and trial number.

## 4. Data availability

The data included in this repository are shared publicly for research, educational, and reproducibility purposes.

No separate access request is required for the files already included in the public repository. If additional non-public data exist, they are not covered by this public release and may require a separate agreement with the data owners or maintainers.

## 5. Privacy and anonymization

The public dataset is intended not to include direct personally identifiable information. Participants are represented using anonymized subject identifiers.

However, because motion patterns can be sensitive and may be used for behavioral or biometric analysis, users of the dataset must:

- not attempt to identify, re-identify, or contact any participant;
- not combine the dataset with other data sources for re-identification;
- not publish individual-level analyses that could reasonably reveal a participant's identity;
- use the data only in ways consistent with ethical research practice.

## 6. Permitted uses

The data may be used for:

- reproducing the experiments described in the repository;
- developing and evaluating machine learning models for task recognition, user identification, or user authentication;
- academic research, teaching, benchmarking, and non-commercial or commercial experimentation permitted by CC BY 4.0;
- creating derived features, models, or aggregate results, provided this policy is respected.

## 7. Restrictions

Users must not:

- use the data to identify or profile real individuals outside the stated research context;
- use the data for surveillance, discriminatory, deceptive, or harmful applications;
- redistribute modified versions of the dataset without clearly documenting the changes made;
- claim that derived results are endorsed by the original authors unless explicit permission has been granted;
- remove or obscure attribution, license, or policy information associated with the dataset;
- apply legal terms or technological measures that restrict others from exercising rights allowed by CC BY 4.0.

## 8. Redistribution

Redistribution of the original public dataset is allowed only when this policy, the CC BY 4.0 data license, the repository software license where applicable, and appropriate attribution are preserved.

Redistribution of derived datasets is allowed when:

- derived files do not increase privacy or re-identification risk;
- transformations are documented clearly;
- the original repository is cited;
- the CC BY 4.0 license is respected;
- this policy or an equivalent privacy-preserving policy is included.

## 9. Citation and attribution

When using the data or code in academic work, publications, reports, or public demonstrations, cite the repository and any accompanying paper specified by the authors.

If a formal paper citation is provided by the maintainers, use that citation in addition to the repository attribution.
